F29AI1 – week 5 Lab

Activity 1 – Running the Search Demonstrator

We will run the search demonstrator that allows you to see how different types of search work

  1. Create a directory in your home area for this activity (e.g mkdir search; cd search)
  2. Copy the search files (especially SampleProblems.txt and the whole directory Saves) to this directory:

cp -r /u1/cs3/public/search/* .

  1. Execute the search.jar file that runs the demonstration like this:

java -jar /usr/java/jdk1.6.0_14/jre/lib/ext/search.jar

  1. From the File menu, choose Load Sample Graph, and from the submenu, choose Simple Tree Graph
  2. Look at the Search Options menu to see what type of search the system is set up for.
  3. Now click on the Solve tab. By clicking on the Step icon top-left, you can see where the search will go with the diamond-shaped node marked ‘G’ the goal of the search.
  4. Try each different search and make sure you understand why each search choice is made:

a)    What is the difference between Lowest Cost First and Best first?

b)   Which search approach finds the solution in fewest steps? Why?

  1. Now load the Cyclic graph example and try the searches again.

a)    Which searches get completely stuck? Why?

b)   Which don’t get stuck? Why?

c)    What does turning on ‘loop detection’ do?
